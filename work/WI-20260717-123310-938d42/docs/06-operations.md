# 運用設計・運用引継ぎ

## SLO・SLI・エラーバジェット

常時serviceなし。campaign reproducibility、unexpected harmful action 0、catalog drift 0をquality targetとする。

## 監視・ログ・アラート

report artifact、test output、validator、PR/CI、source linksを監視。attack budgetとutilityをfinding countと分離する。

## Runbook・インシデント対応

unsafe preconditionは実行せずsynthetic substitute。unstable judgeはdeterministic/humanへescalate。source driftは別更新work item。

## バックアップ・復旧・DR

Gitでskill/docsを復旧。sensitive raw artifactはGit外でtarget policyに従う。

## 容量・コスト・依存先

risk-based portfolioとbudget、model generationは必要時だけ。全techniqueの機械適用を避ける。
