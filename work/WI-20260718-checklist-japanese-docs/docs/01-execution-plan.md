## 許可しない操作

- Pull Requestのマージ、デプロイ、リリース、外部サービス設定変更
- 添付PDFまたは権利不明な資料の公開・再配布
- 認証情報や利用者データの収集・保存
- 本計画に無関係なリポジトリ履歴の変更、破壊的な削除

## 既定の判断

| 判断点 | 追加確認せず採用する既定値 | 根拠 |
|---|---|---|
| 標準の位置付け | 「準拠」ではなく「参照・整合」 | SWEBOK/WAFは案件文脈で適用する知識・判断枠組みであるため |
| 重要度 | 基準重要度は助言、案件重要度をゲートに使う | 重要度インフレと一律適用を避けるため |
| 重複統制 | Core→Cloud Common→採用ベンダー差分→条件付きの順で合成 | 同一リスクの二重計上を避けるため |
| Fail | 是正後の再確認Passまでゲートを通さない | 単一承認の権限境界を維持するため |
| 文書言語 | 識別子・製品名を除き日本語 | 利用者向け文書の統一要望に従うため |
