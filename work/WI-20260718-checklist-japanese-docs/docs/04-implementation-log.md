## セキュア実装・依存関係

新規の本番依存関係は追加していない。既存Python標準ライブラリと固定済み開発依存を使用し、外部添付はコードとして実行しない。SWEBOKはSHA-256一致を必須にし、PDF本体を`.gitignore`対象へ置く。証跡はリポジトリ内の相対パスを要求し、承認とイベントはダイジェスト・連鎖ハッシュで改変を検出する。

## 設計との差異

設計との差異はない。既存の承認済みwork itemをスキーマ2へ移行すると承認ダイジェストが変わるため、本作業記録は互換読取されるスキーマ1のまま維持し、新規work itemからスキーマ2を適用した。これは詳細設計の移行方針どおりである。
