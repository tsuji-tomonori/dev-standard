# リスク・セキュリティ設計

## 資産とデータ分類

| 資産 | 分類 | CIA要求 | 所有者 |
|---|---|---|---|
| 添付SWEBOK PDF | 著作物・外部入力 | 完全性: SHA-256固定、機密性: 公開物、可用性: 公式入手先 | IEEE Computer Society |
| 出典台帳 | 公開メタデータ | 完全性: スキーマ・公式ドメイン・鮮度検査 | ガバナンス所有者 |
| 要件正本 | 公開仕様 | 完全性: revision競合検査・原子的置換 | 要件所有者 |
| チェック実施結果 | 開発記録 | 完全性: 連鎖ハッシュ・証跡検査、機密性: 公開可能な証跡だけを格納 | レビュー担当 |
| GitHub Pull Request | 公開変更 | 完全性: commit・CI・レビュー履歴 | リポジトリ所有者 |

## 脅威モデル

| 境界・フロー | 脅威または非該当根拠 | 影響×可能性 | 統制 | 対応責任者 | 残余リスク |
|---|---|---|---|---|---|
| 外部添付→ローカル検証 | 改ざん・不正構造・権利不明資料の再配布 | 高×中 | 実行しない読取、版・ページ・ハッシュ検証、PDFをGit対象外に維持 | 実装担当 | 低: 元資料が将来入手不能になる可能性 |
| 公式Web→出典台帳 | URL更新や内容改訂により根拠が変わる | 中×高 | 公式HTTPSドメイン、参照日、更新間隔、版・差分を記録 | 出典台帳所有者 | 中: 継続更新型資料は同一URLで変わり得る |
| 正本→派生物 | 手編集や生成器の欠陥で不整合になる | 高×中 | 厳格スキーマ、決定的生成、バイト差分検査 | 正本所有者 | 低: 生成器と検査の共通欠陥 |
| チェック結果→品質ゲート | N/A乱用、偽証跡、重要度過小評価 | 高×中 | 具体理由、到達性、案件重要度根拠、レビュー担当・日付、履歴、再確認を必須化 | レビュー担当 | 中: 内容の妥当性は批判的レビューを要する |
| ローカル→GitHub | 未検証変更や機微情報の公開 | 高×低 | stage対象確認、秘密情報検査、全テスト・CI、PR差分確認 | リポジトリ所有者 | 低: 接続先権限の誤設定 |

## プライバシー・法令・責任あるAI

個人データやAI推論を扱わない。添付資料は著作権条件が明確でないため本体を再配布しない。AIレビュー項目はResponsible AI、Machine Learning、Generative AI、Agentic AIの公式資料を用途別に選択し、法令・組織方針・影響範囲から適用性を判断する。明示同意、マルチリージョン等を一律要求しない。

## リスク受容候補

現時点で受容が必要な未是正リスクはない。継続更新型のクラウド資料は固定内容を完全再現できないため、版がない場合は参照日・変更確認日・前版との差分を残し、更新間隔内に再確認する。案件Criticalの未解決Failは受容候補にせず、是正と再確認を必須とする。
