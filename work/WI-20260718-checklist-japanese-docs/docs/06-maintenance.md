# 保守・変更・廃止計画

## 所有者と保守体制

リポジトリ所有者が要件正本、出典台帳、チェック項目マスターを保守する。AIは会話から版付き変更セットを作成し、生成・テスト・監査・PRまで実行する。内容判断が必要な例外はwork itemへ根拠と責任者を残し、正本への変更は安定IDで追跡する。

## 依存関係・脆弱性・更新方針

Python、openpyxl、Ruff等の開発依存は`requirements.txt`とCIで管理する。出典台帳は継続更新資料の再確認期限を検査し、変更時は正式名称、版、参照日、差分、該当チェックIDを同一PRで更新する。依存脆弱性や公式資料の重大変更は通常のwork itemとして要件・影響・テストを記録する。

## 互換性・移行方針

安定した要件ID、チェックID、CLIサブコマンド、標準ディレクトリを互換境界とする。実施結果スキーマ1は読取を継続し、新規work itemをスキーマ2で作る。要件・出典・ワークブックは生成器を介して移行し、承認済み文書のダイジェストを暗黙に変更しない。

## EOL・データ廃棄

要件は物理削除せず`retire`で履歴を保持する。出典廃止時は置換先、廃止日、影響範囲を台帳へ記録する。work記録は監査証跡としてGit履歴に残す。ローカル添付PDFと一時描画物はGit管理外で、検証後に削除できる。秘密情報や個人データを保存しない。
