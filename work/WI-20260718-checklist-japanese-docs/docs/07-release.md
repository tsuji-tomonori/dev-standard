# リリース判定

## リリース対象

要件・出典正本、チェックリスト生成器と成果物、実施結果ゲート、日本語文書、検証Skill、回帰テスト、完了したwork item記録を一つのPull Requestとして公開する。添付SWEBOK PDFと一時描画物は対象外とする。

## 受入条件の充足

- SWEBOK v4.0a、全18KA、参照・整合の位置付け、出典差分を追跡できる。
- 適用性、案件重要度根拠、Pass/N/A/Fail、証跡、是正、期限、レビュー、再確認、履歴をゲートが検証する。
- 指摘されたクラウド・セキュリティ・AI項目を条件付きかつ原子的にした。
- 1,740件、65追加記録、有界数式、全24シート描画、数式エラー0を確認した。
- 利用者向けMarkdownと生成文書を日本語へ統一した。
- ローカル全検証とwork item監査が成功した。

## 未解決Issue・例外承認

未解決Issue、案件Critical Fail、期限超過、例外承認、リスク受容はない。継続更新型資料の内容固定不能は、変更確認日と差分を保守する運用上の残余リスクとして明示済みである。

## デプロイ・ロールバック

外部デプロイは行わない。Pull Requestをレビュー後に通常のsquash/mergeでmainへ反映する。問題時は該当変更をGit revertし、正本から派生物を再生成する。履歴を書き換える破壊的rollbackは行わない。

## 利用者・運用者への通知

Pull Request本文に、位置付け変更、実施記録スキーマ、v4.0a固定ハッシュ、日本語文書、移植時のコピー範囲、検証結果、残余リスクを日本語で記載する。

## Go / No-Go判断

ローカル統合検証とwork監査が成功したためPull Request作成はGoとする。mainへのマージはユーザーが行うため、本作業の権限外である。PR CIにFailが出た場合はNo-Goへ戻し、同じ範囲内で修正後に再判定する。
