# 振り返り・改善

## 成果と計画差

計画した要件、出典、ワークブック、ゲート、文書、Skills、テストを同一変更として整合させた。現行ワークブックは旧添付版より実施列や個別修正が進んでいたため、全面置換せず既存生成器を強化した。計画外に、Excel全行式が描画時に大きなメモリを消費する事実と、サマリだけに旧SWEBOKファイル名が残る不整合を発見し、生成器側で修正した。

## 有効だった統制

固定ハッシュで参照版を確定したこと、正本と派生物を分離したこと、work item自身へチェックリストを適用したこと、全シートを独立ランタイムで描画したことが有効だった。特に有界数式への変更で、従来失敗したワークブック読込・描画が完了した。

## 手戻り・ゲート失敗・見逃し

初回描画で全列式によるメモリ問題があり、有界範囲へ変更した。再描画時にサマリの対象ドキュメント列だけ旧`swebok-v4.pdf`表記が残ることを検出した。レビューシートだけを正規化対象にしていたことが見逃しの原因である。

## 根本原因

検証対象を「各レビューシートの項目」と「生成シートの表示」に分けて考え、同じ参照名がサマリにも複製されている依存関係を明示していなかった。Excelの最大行まで式を配置する設計は、計算ソフト上の動作だけを見て外部読込・描画の資源制約を受入条件にしていなかった。

## 次回確認方法

チェック項目追加・出典更新時に、正本差分検査、旧参照名走査、件数・ID・原子性テスト、数式エラー走査、全シート描画、work item監査を同じPRで実行する。継続更新型資料は台帳の再確認期限に従い公式一次情報を比較する。
