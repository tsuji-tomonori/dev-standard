# セキュリティ・責任あるAIレビュー結果

## 対象と方法

変更された要件・出典・ワークブック生成・CLI入力・work記録・公開差分を対象に、脅威モデル、入力境界、権限境界、証跡改変、著作物再配布、依存関係を確認する。実装レビュー、異常系単体テスト、Git追跡状態、添付ハッシュ、Skillsの読取専用レビュアー設定を証跡とする。

## 脅威・脆弱性検証

- 外部PDF/Excelは構造読取だけで、マクロ・埋込コード・外部リンクを実行しない。
- SWEBOKの予期しない版・内容はSHA-256不一致で停止する。
- JSONの未知値、必須属性欠落、証跡不達、期限不正、再確認欠落を拒否する。
- 読取専用reviewer、最大3 thread、深さ1、軽量モデル指定をリポジトリ検証で強制する。
- 新規ネットワークサービス、認証面、秘密情報、実行時依存は追加していない。

## データ・プライバシー検証

個人データは処理・保存しない。添付SWEBOKは公開資料だが再配布権を仮定せず、版・正式名称・公式URL・SHA-256だけを公開成果物へ残す。work記録と証跡には公開可能な開発情報だけを記載し、秘密情報がないことをGit差分で確認する。

## AI固有評価

AIモデル、推論、学習データ、エージェント実行は本変更に含まれないため運用統制はN/Aである。チェック項目マスターとしては、AWSのResponsible AI、Machine Learning、Generative AI、Agentic AI Lensを別出典として登録し、AI種別ごとの条件付き適用、比例的な人間監督、追跡、明示的契約、データ由来を原子的項目へ対応付けた。

## 残余リスク・例外

継続更新型Web資料は同一URLでも内容が変わり得るため、変更確認日と前版との差分で定期再確認する。生成器と検査に共通欠陥がある可能性は、添付旧版との比較、独立したExcel読込・描画、回帰テストで低減した。未解決のセキュリティFail、例外承認、リスク受容はない。
