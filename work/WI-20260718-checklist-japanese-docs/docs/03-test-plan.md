# テスト計画

## 品質リスク

- 出典版・添付物が誤っていても生成できる。
- 一つの項目へ複数統制が再結合され、部分Passが可能になる。
- N/A、Fail、再確認の記録不足をゲートが見逃す。
- Excel全行数式や不正参照で読込・描画が失敗する。
- 日本語文書と正本派生物が乖離する。
- 既存CLI、承認ダイジェスト、実施結果スキーマ1の互換性を壊す。

## セキュリティ・AI評価

外部添付を実行せず、PDFの版・ページ・SHA-256とExcel構造だけを読む。秘密情報や個人データを扱わないこと、PDFをGit管理しないこと、証跡パスがリポジトリ外へ逸脱しないことを確認する。AI機能は実装しないためモデル評価はN/Aだが、Responsible AI、Machine Learning、Generative AI、Agentic AIの出典分離と条件付き適用を回帰検査する。
