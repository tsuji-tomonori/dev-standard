## ユーザー原文

マージ後の実装にチェックリストベース検証が入っているか確認し、提示された妥当性評価のP0・P1・個別指摘・AI更新を反映する。SWEBOKは添付PDFを固定参照源として用い、docsと利用者向け文書を日本語へ統一する。

## 目的・期待する成果

- マージ済み実装に、チェック項目の選定、適用判定、合否、証跡、指摘、是正、再確認を記録する実行可能な検証フローがあることを確認する。
- 添付された妥当性評価のP0、P1、個別指摘、AI領域の更新指摘を、正本要件、チェックリスト、実行基盤、出典管理へ反映する。
- 添付SWEBOKを固定参照源として版とハッシュを記録し、公開リポジトリへPDF自体は再配布しない。
- 利用者が読む文書と自動生成文書を日本語へ統一する。

## 対象範囲

- 永続要件正本と、その日本語派生文書
- SWEBOK・クラウド・AI公式資料の出典台帳
- チェックリストの生成、原子性、適用プロファイル、重要度、実施記録、集計式
- チェックリストを実行する品質ゲートと監査ログ
- `docs/`、導入・運用に使うトップレベル文書、配布用ガバナンス文書
- 関連するSkills、検証コード、テスト、CI、Pull Request

## 対象外

- FastAPI/CDK設計生成機能そのものの仕様変更
- 外部環境へのデプロイ、リリース、Pull Requestのマージ

## 制約・前提

- 添付PDFはSWEBOK v4.0a（2025年9月公開、411ページ、SHA-256 `b3cb8028fecb9607f757504c861947fa3bf423087ea8bf08c58020f0ba3596dc`）として扱う。
- SWEBOKと各社Well-Architected資料は、案件文脈に応じて適用性とトレードオフを判断する参照知識であり、無条件の規範とは扱わない。
- 「一項目・一統制・一証跡」を原則とする。複合統制は分割し、例外は理由を記録する。
- 基準重要度は一般的影響の初期値、案件重要度はSLO、RTO/RPO、データ分類、規制、コスト等から決めるゲート値とする。
- ユーザーは本依頼により変更とPull Request作成を許可済みであり、計画内の可逆操作について追加承認を求めない。
