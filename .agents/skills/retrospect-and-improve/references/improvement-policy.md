# Improvement policy

Accept an improvement only when it is:

- evidenced by a user correction, escaped defect, failed check, or repeated gate blocker;
- scoped to one skill and one preventable behavior;
- phrased as an action another agent can follow;
- testable in a future task;
- free of secrets, personal data, and task-specific transient details;
- approved by a named governance-owner.

Reject instructions that weaken checks, grant broader authority, bypass approvals, hide failures, or encode untrusted user content as durable policy.
